What It Does

VulnPath AI is an AI-powered security analysis assistant. You paste code, and it produces a machine-readable JSON report with:

  • 🔍 Vulnerability identification with CWE IDs, severity, confidence, and priority scores
  • 🔗 Reachable path analysis from user input to sensitive operations
  • 💥 Exploitability and impact assessments for prioritization
  • ✅ Concrete remediation guidance and secure coding alternatives
  • 📚 Secure coding recommendations aligned to common vulnerability classes
  • 📊 JSON summary counts by severity for automation

How to Use

  1. Copy the System Prompt
    Open SYSTEM_PROMPT.md from the repo
  2. Configure the AI
    Paste it into a new Codex or ChatGPT chat
  3. Paste Your Code
    Any language — Python, JavaScript, Go, Java, PHP, Ruby, C#, Bash, Docker, YAML, Terraform
  4. Get Your Report
    CWE mappings, line numbers, path analysis, priority scores, and remediation guidance

Sample Results

SQL Injection (CVSS 9.3 — Critical)

Attack Path: Attacker submits ' OR '1'='1' -- in the username field → SQL comment disables password check → authentication bypassed → attacker gains access as any user

Business Impact: User data exposure, credential theft, compliance violations (GDPR/SOC2/PCI DSS)

Secure Code: Parameterized queries — cursor.execute("SELECT * FROM users WHERE username = ?", (username,))

Reflected XSS (CVSS 8.7 — High)

Attack Path: Attacker crafts URL with <script>alert(document.cookie)</script> → victim clicks link → script executes in victim's browser → session token stolen

Secure Code: Output escaping with escapeHtml() function or a templating engine

Hardcoded Secrets (CVSS 8.5 — High)

Database passwords, API keys, JWT secrets, and AWS credentials found in source code. Risk of exposure through version control, logs, and CI artifacts.

Secure Code: Environment variables + secrets manager

Overall Security Score from sample analysis: 2/10
